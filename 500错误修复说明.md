# 500错误修复说明

## 🔍 问题诊断

### 问题1：数据库名配置不一致（已修复）

**问题描述：**
- `application.yml` 中配置的数据库名：`chronic-disease-analysis`
- `init.sql` 中创建的数据库名：`chronic_disease_db`

**影响：**
- 应用启动时无法连接到正确的数据库
- 所有数据库操作都会失败，导致500错误

**修复方案：**
修改 `backend/src/main/resources/application.yml` 第16行：

```yaml
# 修复前
url: jdbc:mysql://localhost:3306/chronic-disease-analysis?...

# 修复后
url: jdbc:mysql://localhost:3306/chronic_disease_db?...
```

---

### 问题2：Service层查询类型不匹配（已修复）

**问题描述：**
在 `ChronicDiseaseDataServiceImpl` 中，多个方法使用 `String` 类型的 `diseaseCode` 参数直接查询 `Long` 类型的 `diseaseId` 字段：

```java
// 错误示例
wrapper.eq(ChronicDiseaseData::getDiseaseId, diseaseCode);  // 类型不匹配！
```

**影响：**
- MyBatis-Plus无法正确转换类型
- SQL查询执行失败，抛出异常
- 前端收到500错误

**修复方案：**
在 `ChronicDiseaseDataServiceImpl` 中添加了类型转换逻辑：

```java
// 新增方法：根据疾病编码获取疾病ID
private Long getDiseaseIdByCode(String diseaseCode) {
    if (diseaseCode == null || diseaseCode.isEmpty()) {
        return null;
    }
    LambdaQueryWrapper<DiseaseType> wrapper = new LambdaQueryWrapper<>();
    wrapper.eq(DiseaseType::getDiseaseCode, diseaseCode);
    DiseaseType diseaseType = diseaseTypeMapper.selectOne(wrapper);
    return diseaseType != null ? diseaseType.getId() : null;
}

// 修改后的查询示例
@Override
public List<Map<String, Object>> getPrevalenceByProvince(String diseaseCode, Integer year) {
    LambdaQueryWrapper<ChronicDiseaseData> wrapper = new LambdaQueryWrapper<>();
    if (diseaseCode != null) {
        Long diseaseId = getDiseaseIdByCode(diseaseCode);  // 先转换
        if (diseaseId != null) {
            wrapper.eq(ChronicDiseaseData::getDiseaseId, diseaseId);  // 再查询
        }
    }
    List<ChronicDiseaseData> list = list(wrapper);
    return list.stream().map(BeanUtil::beanToMap).collect(Collectors.toList());
}
```

**受影响的方法（已全部修复）：**
1. `getPrevalenceByProvince()` - 各省份患病率统计
2. `getPrevalenceTrend()` - 患病率趋势数据
3. `getPrevalenceByAgeGroup()` - 各年龄段患病率分布
4. `getTop10Prevalence()` - 全国患病率Top10
